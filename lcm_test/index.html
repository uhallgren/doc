Introduction

This project requires some build tools and libraries for a build to succeed. Below is a quick setup to be able to build the project. Other pages may in the future give more information about the various tools. The purpose is now to show first how to quickly build the project.

The impatient and brave may set up the needed tools by running the provided shell script. 

./setup_project.sh

The following sections explain step by step what is done in the shell script.

Build tools

GNU C and C++ are of course needed. 

sudo apt install build-essential

Meson is a rising star when it comes to systems that support building C/C++. It uses Ninja for the actual build. 

sudo apt install meson
sudo apt install ninja-build

Clang-Format formats the source code so that it has a consistent format. This does also help to avoid differences in pull requests that cause unneeded work. Clang-Tidy is a clang-based C++ “linter” tool. Its purpose is to provide an extensible framework for diagnosing and fixing typical programming errors, like style violations, interface misuse, or bugs that can be deduced via static analysis. 

sudo apt install clang-format
sudo apt install clang-tidy

Doxygen is the de facto standard tool for generating documentation from annotated C++ sources. Doxygen can make dependency graphs and this requires the program dot. The program is provided in the package graphviz. 

sudo apt install doxygen
sudo apt install graphviz

Dependencies

The project needs some external libraries. These are not built as part of the project to keep the build setup simple and also to make the build process as fast as possible. Some of these dependencies will be installed from the source code. They do thus depend on some additional tools. 

sudo apt install cmake
sudo apt install pkg-config

GoogleTest is Google’s C++ testing and mocking framework. 

sudo apt install libgtest-dev

JSON for modern C++ provides an intuitive syntax that makes JSON feels like a first-class data type. It provides trivial integration as the whole code consists of a single header file. This is important especially as the project needs to support cross-compiling. It needs to be installed from the source code. It needs CMake to build the installation.

pushd /tmp
git clone https://github.com/nlohmann/json.git
cd json
git checkout v3.11.2
mkdir builddir
cd builddir
cmake -DCMAKE_BUILD_TYPE=Release -DJSON_BuildTests=OFF  -G Ninja ..
sudo ninja install
popd

spdlog is a header-only or compiled C++ logging library. 

pushd /tmp
git clone https://github.com/gabime/spdlog.git
cd spdlog
git checkout v1.11.0
mkdir builddir
cd builddir
cmake -DCMAKE_BUILD_TYPE=Release -G Ninja ..
sudo ninja install
popd

Build the project

If all the previous build tools and dependencies are installed then the project can easily be built. It is assumed that the project is checkout from the source and that the working directory is the source root of the project. 

meson setup builddir
meson compile -C builddir

If you would like to know what is going on during the build process then it is possible to use Ninja in verbose mode. 

ninja -C buildir clean
ninja -C buildir -v

It is also possible to build the documentation. This will produce easy-to-read web pages. 

ninja -C builddir doc/html
xdg-open builddir/doc/html/index.html
